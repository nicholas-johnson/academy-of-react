# Quest 1: Auth System

Create AuthContext tracking logged-in student (name, house, role: student/professor/headmaster). Login/logout functionality. Components display different content based on auth status.

## Requirements (TypeScript)
- AuthContext with user state
- Login function (sets user)
- Logout function (clears user)
- useAuth custom hook
- Multiple components consume context
- Conditional rendering based on role
- TypeScript interfaces for User

## Acceptance Criteria
- [ ] Context created and typed
- [ ] Provider wraps app
- [ ] Login/logout work
- [ ] Components use useAuth
- [ ] Role-based content shows
- [ ] Full TS type safety
